    Class Overseer

    Central orchestrator for the KalshiBot Overseer system that manages WebSocket connections, handles real-time market data events, coordinates periodic data fetching, and maintains system health monitoring. This class serves as the main entry point for overseer operations, integrating with WebSocket clients, database services, and SignalR hubs to provide comprehensive monitoring and control of the trading bot ecosystem.

    Key features:

    • Configurable API fetch intervals (default: 10 minutes)
    • Configurable system info logging intervals (default: 1 minute)
    • Performance metrics collection for WebSocket events and API operations
    • Input validation for event data processing to prevent null reference exceptions
    • Progress reporting for long-running API fetch operations
    • Batch processing for brain persistence logging (default batch size: 50)
    • Configurable SignalR broadcast batch sizes (default: 10)
